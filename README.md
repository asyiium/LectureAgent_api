# Конспект-хелпер (Lecture Agent), backend

## Краткое описание

Асинхронный бэкенд-сервис на базе FastAPI для удобной работы с текстовыми файлами и конспектами. Сервис поддерживает чат с большими языковыми моделями (GigaChat), потоковую генерацию ответов (Server-Sent Events), работу с векторными базами данных (Chroma) для поиска по загруженным документам (PDF, PPTX, TXT, аудио), а также интеграцию с внешним поиском (Tavily) и создание тестов на основе предоставленных материалов.

## Возможности

-   **Чат с LLM**: Поддержка диалогов с потоковой выдачей ответов через SSE.
-   **Гибкая маршрутизация**: Автоматическое определение намерения пользователя (`general` или `test`) роутер-моделью.
-   **Работа с RAG (Retrieval-Augmented Generation)**: Индексация и поиск информации по загруженным документам в векторной БД Chroma.
-   **Поддержка медиафайлов**: Загрузка и обработка различных форматов:
    -   Текст: `.txt`, `.md`
    -   Документы: `.pdf`, `.pptx`, `.ppt`
    -   Аудио: `.mp3`, `.ogg`, `.wav`, `.flac` (с транскрибацией через Salute Speech)
-   **Поиск в интернете**: Использование Tavily Search API для получения актуальных данных.
-   **Генерация тестов**: Автоматическое создание тестов с правильными и неправильными вариантами ответов по запросу пользователя.
-   **Управление чатами**: Создание, удаление, просмотр истории и списка чатов пользователя.
-   **Асинхронность**: Полная асинхронная обработка запросов (FastAPI + `async/await`).

## Технологический стек

-   **Web-фреймворк**: FastAPI
-   **ORM / База данных**: Redis (хранение метаданных и истории)
-   **Векторная БД**: Chroma (локальное хранение эмбеддингов)
-   **LLM & Embeddings**: GigaChat (API)
-   **Фреймворк агентов**: LangChain (LangGraph Agent + Tools)
-   **Транскрибация аудио**: Salute Speech API (Сбер)
-   **Поиск в интернете**: Tavily API

### Архитектура

1.  **main.py**: Точка входа (API эндпоинты: `/ask`, `/create_chat`, `/add_media` и др.).
2.  **llm_service.py**: Ядро логики. Управление LLM, агентами, векторизацией, инструментами (Tools) и стримингом.
3.  **chat_manager.py**: Бизнес-логика управления чатами. Координирует работу Redis и LLM Service.
4.  **redis_client.py**: Абстракция для работы с Redis (история, метаданные, пользователи).

## Быстрый старт полного проекта

### Необходимо наличие:

-   Python 3.10 или выше
-   Установленный и запущенный Redis сервер (легко реализуется через Docker-compose)
-   API ключи от GigaChat, Tavily, Salute Speech (для аудио)

### Установка и запуск

1.  **Клонируйте репозиторий и перейдите в директорию:**
    ```bash
    git clone <your-repo-url>
    cd <project-folder>
    ```
    
2. **Создайте файл .env по формату:**
   ```
     CA_BUNDLE_PATH=/path/to/your/ca_bundle.pem # путь к сертификату Минцифры
   
     TEXT_LLM_API_KEY=[ключ-от-Гигачата] 
     AUDIO_LLM_API_KEY=[ключ-от-Салюта]
     TAVILY_API_KEY=[ключ-Тавили]
   
   ```
   
3. **Запускайте проект ([frontend находится здесь](https://github.com/asyiium/LectureAgent_front)) через docker-compose или индивидуально каждый компонент, предварительно установив зависимости в корне репозитория (API, front, redis)**
